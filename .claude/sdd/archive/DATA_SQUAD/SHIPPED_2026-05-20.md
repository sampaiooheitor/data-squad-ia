## Summary

O Data Squad é um sistema multi-agente Python que simula um engenheiro de dados autônomo end-to-end. A partir de um único comando (`python -m src.orchestrator`), o sistema: gera um dataset CSV fictício com tema aleatório via Claude API, submete ao Google Forms via Playwright, infere o schema, gera regras de qualidade de dados e classificação de PII em paralelo, produz notebooks Databricks (Bronze + Silver) com DQ e masking inline, abre um PR no GitHub com todos os artefatos, executa os notebooks no Databricks via Commands API, e notifica o Slack com o resumo do run. O ciclo completo ocorre sem nenhuma intervenção humana.

## Lessons Learned

### Process

- **Brainstorm antes de definir poupou rework.** A sessão de brainstorm (Phase 0) já respondeu todas as dúvidas de escopo — o DEFINE teve clareza 15/15 sem nenhuma pergunta adicional. Vale sempre rodar o brainstorm antes do define.
- **Desvio documentado no BUILD_REPORT é ativo, não passivo.** O Reporter Agent rodando depois do GitHub PR (não em paralelo como no design) foi um desvio que identificamos rápido porque o design estava claro. Sem o DESIGN, esse bug de lógica seria descoberto só na execução.
- **Ciclo Brainstorm → Define → Design → Build em 2 dias** é viável para sistemas desta complexidade com o SDD workflow.

### Technical

- **Claude API sem tool_use é mais previsível para demos.** A escolha de usar `messages.create()` simples (sem tool_use) em todos os agentes tornou o fluxo 100% determinístico — Claude retorna JSON, o orquestrador age. Nenhuma surpresa.
- **JSON Schema no system prompt aumenta conformidade da saída.** Incluir `model.model_json_schema()` no system prompt reduziu a chance de JSON inválido. Combinado com `model_validate_json()`, falhas são rápidas e claras.
- **`asyncio.gather` + `run_in_executor` é o padrão correto** para misturar agentes async (Claude API) com ferramentas síncronas (Playwright, requests, PyGithub). Evita bloquear o event loop sem precisar de multiprocessing.
- **`copy.deepcopy(ctx)` para fases paralelas** funciona perfeitamente porque RunContext contém apenas primitivos e modelos Pydantic — todos copiáveis. Fases paralelas escrevem campos disjuntos, sem race conditions.
- **Databricks Free Edition: sem Jobs API.** Confirmado durante o design — a solução via Workspace API 2.0 + Commands API 1.2 funciona, mas requer criar e destruir um contexto de execução por run.

### Communication

- **O desvio de paralelismo (Reporter depois do GitHub) foi detectado no design**, não no build. O DESIGN explicitou que o Reporter precisa do `pr_url`, o que tornou óbvio que não poderia rodar em paralelo com o GitHub Agent.
- **Constraints explícitas no DEFINE (Databricks Free Edition, sem Jobs API)** guiaram diretamente uma decisão de arquitetura no DESIGN. Sem essa constraint documentada, o design poderia ter usado a Jobs API e falhado silenciosamente.

### Tools & Libraries

- **pydantic-settings** — leitura de `.env` + type hints + defaults em uma única classe. Eliminou todo o boilerplate de `os.getenv()`.
- **PyGithub** — abstração limpa para criar branch + commit + PR. Mais confiável que requests diretos para a GitHub REST API.
- **ruff** — corrigiu 12 de 13 erros automaticamente com `--fix`. Inclua no workflow desde o início.
- **Playwright sync_api** — mais simples para integração com `run_in_executor` do que a API assíncrona nativa, já que o executor isola a chamada em thread separada.

## Recommendations for Future Work

| Área | Recomendação |
|------|--------------|
| v2 — Persistência local | Salvar artefatos em `generated/{run_id}/` antes de commitar no GitHub |
| v2 — Human-in-the-loop | Adicionar etapa de aprovação opcional antes do PR (flag `--review`) |
| v2 — Retry resiliente | Exponential backoff com `tenacity` para chamadas Claude API e Databricks |
| v2 — Memória entre runs | Vector DB (pgvector ou Qdrant) para aprender padrões de DQ recorrentes |
| v3 — Dashboard | Streamlit ou Gradio mostrando progresso em tempo real por fase |
| v3 — Multi-tenant | Auth + isolamento por usuário para produtização |
| Produção | Substituir `asyncio.get_event_loop()` (deprecated em Python 3.12) por `asyncio.get_running_loop()` |
| Produção | Adicionar testes unitários com pytest + respx para agentes (mock Claude API) |
